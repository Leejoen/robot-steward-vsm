# 10. Безопасность

## Модель безопасности MVP

MVP не обрабатывает реальные платежные данные, документы или юридически значимые персональные данные. Основная задача безопасности - не допустить утечки тестовых данных, некорректной роли пользователя, опасных рекомендаций пассажиру и подмены ответа цифровой платформы.

## Активы

| Актив | Риск | Защита MVP |
| --- | --- | --- |
| Данные поездки | Некорректное отображение или подмена | Тестовый справочник и валидация |
| Ответ платформы | Неверный маршрут или подсказка из-за ошибки адаптера | Валидация контракта и безопасный fallback |
| Обращения пассажиров | Утечка описаний проблем | Минимизация данных |
| Административный интерфейс | Несанкционированное изменение модели | Простая авторизация в целевой версии |
| Логи | Попадание лишних данных | Не логировать персональные данные |

## Роли

- Пассажир: создает запросы, смотрит маршрут и информацию по поездке.
- Сотрудник: смотрит и обрабатывает обращения.
- Администратор: меняет mock-ответы платформы и настройки сценариев.

## Угрозы

| Угроза | Митигация |
| --- | --- |
| Ввод некорректной зоны или поезда | Валидация входных данных |
| Попытка открыть административные функции пассажиром | Разделение UI и ролей |
| Выдача опасного маршрута | Проверка предупреждений и ограничений в ответе платформы |
| Хранение лишних персональных данных | Принцип минимизации |
| Ошибка платформы или внешнего источника данных | Явное предупреждение и fallback |

## Правила безопасного ответа

- Не обещать гарантированную посадку на поезд.
- Не скрывать неопределенность, если данные неактуальны.
- Не направлять пассажира через закрытые или недоступные зоны.
- Не подменять отсутствующий ответ платформы собственной догадкой.
- Для нештатных ситуаций предлагать обращение к персоналу.
- Не отображать внутренние ошибки пользователю в техническом виде.

## Усиление после MVP

- Авторизация сотрудников и администраторов.
- Журналирование административных изменений.
- Защита API.
- Политика хранения обращений.
- Проверка соответствия требованиям обработки персональных данных.
