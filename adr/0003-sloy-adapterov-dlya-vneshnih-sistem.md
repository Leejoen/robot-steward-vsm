# ADR-0003. Слой адаптеров для внешних систем

## Статус

Принято.

## Контекст

Целевая система должна получать данные из цифровой платформы умного вокзала и системы заявок персонала. Через платформу робот получает расписание, билетный контекст, маршрут, загруженность и предупреждения. На ранних этапах API платформы может быть недоступен или нестабилен.

Если встроить тестовые данные напрямую в доменную логику, последующее подключение платформы станет сложным и потребует переписывания архитектуры.

## Решение

Все внешние источники данных подключаются через адаптеры. Главный адаптер MVP - `Smart Station Platform Adapter`. В автономном режиме он возвращает заранее подготовленные ответы, а в целевой версии обращается к API цифровой платформы умного вокзала.

## Альтернативы

- Жестко зашить данные в код: быстрее, но ухудшает архитектуру.
- Сразу подключать реальный API платформы: предпочтительно для промышленной системы, но зависит от доступности и стабильности API.
- Хранить все платформенные данные только в базе робота: удобно для демонстрации, но размывает границу ответственности между системами.

## Последствия

- MVP можно запускать без внешней инфраструктуры.
- В документации явно видны границы ответственности.
- Реальную платформу можно добавить позже без изменения сценарного ядра робота.
- Mock-ответы требуется поддерживать в согласованном состоянии.
