# 11. Тестирование

## Стратегия

Тестирование должно подтвердить, что система корректно ведет пассажира по сценариям, использует ответы цифровой платформы и не выдает недостоверную информацию. Главный акцент - на диалоге, адаптере платформы, сервисной эскалации и негативных случаях.

## Уровни тестирования

| Уровень | Что проверяется |
| --- | --- |
| Unit-тесты | Разбор намерений, правила ответа, fallback-поведение |
| Интеграционные тесты | API диалога, адаптер платформы, обращения |
| Сценарные тесты | Полные пользовательские потоки |
| Проверка данных | Полнота mock-ответов платформы |
| Ручная демонстрация | Понятность интерфейса и ответа |

## Матрица критичных проверок

| ID | Сценарий | Ожидаемый результат |
| --- | --- | --- |
| T-01 | Обычный маршрут от входа до платформы | Робот объясняет маршрут, полученный от платформы |
| T-02 | Быстрый маршрут при малом времени до отправления | Робот показывает предупреждение и короткую инструкцию |
| T-03 | Безбарьерный маршрут | Робот явно сообщает, что маршрут учитывает доступность |
| T-04 | Закрытая зона на пути | Робот объясняет изменение и показывает альтернативу из ответа платформы |
| T-05 | Поезд не найден | Робот не придумывает данные и показывает fallback |
| T-06 | Изменение платформы | Робот сообщает об изменении и запрашивает обновленный маршрут |
| T-07 | Невозможный маршрут | Робот возвращает понятную ошибку и предлагает помощь |
| T-08 | Создание обращения | Обращение появляется в журнале |
| T-09 | Аналитика запросов | Счетчики обновляются |
| T-10 | Некорректный ввод | API возвращает валидируемую ошибку |

## Проверка сценариев платформенного адаптера

Для mock-платформы требуется подготовить набор фиксированных ответов:

- обычный маршрут;
- маршрут с альтернативой;
- маршрут с закрытой зоной;
- безбарьерный маршрут;
- маршрут при перегруженной зоне;
- ответ "маршрут недоступен";
- ответ "поезд не найден";
- ответ "платформа изменилась".

## Критерии приемки MVP

- Все обязательные сценарии выполняются на тестовой модели.
- Время ответа сценарного слоя робота без внешней платформы меньше 1 секунды.
- Система объясняет маршрут шагами.
- Негативные сценарии не приводят к некорректным рекомендациям.
- Список рисков и ограничений явно отражен в документации.
