# Интеллектуальная система робота-стюарда для сопровождения пассажиров на вокзале ВСМ

Архитектурная документация первой итерации для прототипа системы робота-стюарда на вокзале ВСМ.

Система рассматривается как программный прототип робота-стюарда: пассажирский интерфейс, диалоговое сопровождение, информационная поддержка по поездке, объяснение маршрута, фиксация обращений и базовая аналитика взаимодействий. Физическая робототехническая платформа, компьютерное зрение и полноценный голосовой интерфейс вынесены за границы MVP.

В целевой экосистеме робот-стюард не является владельцем цифрового пространства вокзала. Источником карты, расписания, рейсов, базовых маршрутов и событий выступает цифровая платформа умного вокзала. Робот-стюард использует ее API как специализированный пользовательский канал и превращает платформенные данные в понятное сопровождение пассажира.

## Статус

Документация описывает целевую архитектуру MVP. Часть интеграций с реальными вокзальными системами заменяется тестовыми адаптерами и имитационными данными, чтобы прототип мог работать автономно до подключения к промышленным API.

## Основные решения первой итерации

- MVP реализуется как веб-прототип или симуляционная система, а не как физический робот.
- Ядро системы - сценарии взаимодействия робота с пассажиром и адаптация платформенных подсказок под человеческий диалог.
- Сопровождение пассажира делится на навигационное, информационное и сервисное.
- В целевой версии маршруты и данные поездки поступают из цифровой платформы умного вокзала.
- В автономном MVP эти данные имитируются локальными адаптерами, чтобы система могла демонстрировать ключевые сценарии без подключения к внешней платформе.
- Архитектура разделяет интерфейс робота, сценарный backend, адаптер цифровой платформы, сервис обращений и аналитику взаимодействий.

## Открытые вопросы

- Какие данные реально доступны от вокзальных систем ВСМ: расписание, путь, зона посадки, изменения, статусы досмотра.
- Каким будет контракт API между цифровой платформой вокзала и роботом-стюардом.
- Достаточен ли для MVP веб-прототип или требуется макет интерфейса на терминале робота.
- Какие метрики качества сопровождения должны использоваться для оценки прототипа.
- Какой уровень детализации безопасности нужен для MVP.
